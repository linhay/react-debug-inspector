# Inspector 扩展复制功能规格（BDD）

## 背景
为 `react-debug-inspector` 的检查模式补充更丰富的复制能力。在保留“点击目标复制 debug id”主路径不变的前提下，支持复制目标文案、图片，以及一次性复制当前目标的完整上下文。

## 验收范围
- 检查模式悬浮命中元素时显示复制操作菜单。
- 默认点击目标元素仍复制 `data-debug` 原值并退出。
- 菜单支持复制 `debug id`、文案、图片、全部信息。
- 图片复制优先尝试图片二进制，失败时自动降级为结构化文本。
- README 与 Demo 页面同步体现新交互。

## 场景 1：默认点击行为保持不变
- **Given** 用户已进入检查模式
- **When** 直接点击一个带 `data-debug` 的目标元素
- **Then** 应复制该元素完整的 `debug id`
- **And** 显示复制成功反馈
- **And** 自动退出检查模式
- **And** 不应触发 React 或宿主应用绑定的业务点击逻辑

## 场景 1.1：检查模式应压制 React 组件点击
- **Given** React 组件在目标元素上绑定了业务 `onClick`
- **And** 用户已进入检查模式
- **When** 用户点击该目标元素以选择 `data-debug`
- **Then** Inspector 应完成复制与退出流程
- **And** React 业务 `onClick` 不应被触发
- **And** 页面状态不应因为该次点击而改变

## 场景 1.2：检查模式应压制更早的鼠标交互链
- **Given** React 组件在目标元素上绑定了 `onMouseDown`、`onPointerDown` 或同类早期交互处理
- **And** 用户已进入检查模式
- **When** 用户点击该目标元素以选择 `data-debug`
- **Then** Inspector 应在复制前先拦截这次选择动作的早期事件
- **And** 业务侧的按下态、预选中态或提前导航逻辑不应被触发

## 场景 2：命中目标时显示复制操作菜单
- **Given** 用户已进入检查模式
- **When** 鼠标悬浮在一个带 `data-debug` 的目标元素上
- **Then** 应显示复制操作菜单
- **And** 菜单至少包含“复制 ID / 复制文案 / 复制图片 / 全部复制”

## 场景 3：复制文案优先取用户可感知文本
- **Given** 用户通过菜单点击“复制文案”
- **When** 当前目标存在 `aria-label`、`value`、`alt`、`title` 或可见文本
- **Then** 应按约定优先级提取最相关文本并复制到剪贴板
- **And** 若没有可复制文本，则回退复制 `debug id`

## 场景 4：复制图片优先写入图片本体
- **Given** 用户通过菜单点击“复制图片”
- **When** 当前目标命中 `<img>` 或背景图元素
- **Then** 应优先尝试将图片 Blob 写入剪贴板
- **And** 成功时显示“已复制图片”反馈

## 场景 5：图片本体不可复制时自动降级
- **Given** 浏览器不支持图片剪贴板写入，或图片因资源/CORS 原因无法转为 Blob
- **When** 用户点击“复制图片”
- **Then** 应降级复制结构化图片信息文本
- **And** 输出至少包含 `url`、`alt`、`title`、`filename`、`debugId`

## 场景 6：全部复制输出结构化文本
- **Given** 用户通过菜单点击“全部复制”
- **When** 当前目标已被命中
- **Then** 应输出 `debug`、`text`、`image` 三段结构化文本
- **And** 内容可直接粘贴给开发者排查问题

## 场景 7：菜单操作不打断检查流程
- **Given** 用户正在检查模式中
- **When** 点击菜单中的任一复制动作
- **Then** 不应因为菜单点击而退出检查模式
- **And** 菜单仍可继续用于后续复制

## 场景 8：Demo 页面可演示图片与文案复制
- **Given** 用户访问 Demo 页面
- **When** 进入检查模式并查看示例区域
- **Then** 页面中应存在可复制文案的文本示例
- **And** 页面中应存在可复制图片的图片示例
