## 2026-03-13 修复 Inspector 点击未完全拦截同级 capture 监听

- 背景：GitHub issue `#4` 反馈在 React 应用中开启检查模式后，点击目标元素时业务 `onClick` 仍可能触发，导致页面跳转或状态变更，影响 `data-debug` 选择。
- BDD：更新 `docs-dev/features/inspector-copy-spec.md`，补充“检查模式点击目标时不得触发 React / 宿主业务点击逻辑”的验收场景。
- TDD：在 `src/runtime.test.ts` 新增回归测试，验证 Inspector 在检查模式下点击目标元素时，会阻断后续注册在 `window` capture 阶段的 click 监听器，覆盖 `stopPropagation` 无法阻断同节点同阶段监听的问题。
- 实现：在 `src/runtime.ts` 增加统一 `suppressEvent` helper，并仅对窗口级检查点击拦截启用 `stopImmediatePropagation()`，避免误伤 Inspector 自身按钮和菜单的内部事件链路。
- 验证：执行 `npm test`，8 个测试文件共 72 个测试全部通过。

## 2026-03-13 修复检查模式未拦截早期鼠标事件链

- 背景：上游 `1.2.1` 已能压住普通 `click`，但真实页面里仍有卡片/列表在 `onMouseDown`、`onPointerDown` 等更早阶段启动交互，导致 inspect 模式下仍可能“选元素即触发业务态”。
- BDD：在 `docs-dev/features/inspector-copy-spec.md` 追加“检查模式应压制更早的鼠标交互链”场景，明确 `onMouseDown`、`onPointerDown`、提前导航/预选中态都不应触发。
- TDD：新增 `src/runtime.test.ts` 回归测试，先打出“检查模式下 mousedown 仍会穿透到后续 window capture 监听”的红灯；同时在 `e2e/inspector.spec.ts` 与 `test-app/src/App.tsx` 增加 Pressable Card 场景，验证真实 React demo 中选择元素不会触发 `onMouseDown`。
- 实现：`src/runtime.ts` 新增 `isInspectorChromeTarget` 与统一的 `handlePointerSuppression`，在检查模式下对非 Inspector UI 的 `pointerdown`、`pointerup`、`mousedown`、`mouseup`、`touchstart`、`touchend` 进行 capture 阶段拦截，并保留原有 click 复制流程。
- 验证：`npm test` 通过（8 files / 73 tests）；`npm run build` 通过；`npm run test:e2e` 通过（15 passed）。
