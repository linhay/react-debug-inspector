## 2026-03-13 修复 Inspector 点击未完全拦截同级 capture 监听

- 背景：GitHub issue `#4` 反馈在 React 应用中开启检查模式后，点击目标元素时业务 `onClick` 仍可能触发，导致页面跳转或状态变更，影响 `data-debug` 选择。
- BDD：更新 `docs-dev/features/inspector-copy-spec.md`，补充“检查模式点击目标时不得触发 React / 宿主业务点击逻辑”的验收场景。
- TDD：在 `src/runtime.test.ts` 新增回归测试，验证 Inspector 在检查模式下点击目标元素时，会阻断后续注册在 `window` capture 阶段的 click 监听器，覆盖 `stopPropagation` 无法阻断同节点同阶段监听的问题。
- 实现：在 `src/runtime.ts` 增加统一 `suppressEvent` helper，并仅对窗口级检查点击拦截启用 `stopImmediatePropagation()`，避免误伤 Inspector 自身按钮和菜单的内部事件链路。
- 验证：执行 `npm test`，8 个测试文件共 72 个测试全部通过。
